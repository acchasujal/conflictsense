## 1. The Fallback Chain
To ensure the demo never crashes on stage (which results in a 0 score), the `ConflictSenseOrchestrator` implements a strict 3-tier fallback chain for every agent call:

- **Tier 1 (Primary):** Azure Foundry IQ `gpt-4o`. Timeout: 10s.
- **Tier 2 (Backup):** Azure Foundry IQ `gpt-4o-mini`. Timeout: 15s.
- **Tier 3 (Mock):** Pre-computed, hardcoded responses from memory. Never fails.

If Tier 3 is activated, the UI must display a `[MOCK MODE]` indicator to remain transparent.

## 2. Confidence Thresholds
LLM outputs inherently contain uncertainty. ConflictSense gates actions based on confidence scores returned by Foundry IQ.

- **High (85% - 100%):** Finding is auto-classified and shown in the main dashboard.
- **Medium (65% - 84%):** Finding is shown but flagged with a `REVIEW REQUIRED` badge.
- **Low (0% - 64%):** Output status is marked `UNCERTAIN`. Finding is routed to a separate human review queue and is NEVER shown on the main dashboard.

## 3. Citation Validation
A conflict is only valid if it is structurally impossible to satisfy multiple policies simultaneously.
- **Rule:** Any conflict output generated by the `ConflictDetector` must contain `>= 2` grounded citations from Foundry IQ.
- **Enforcement:** The orchestrator validates the result. If citations < 2, a `ValidationError` is thrown, triggering the next fallback tier or discarding the finding.
