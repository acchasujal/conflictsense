Foundry IQ workflow — exact query patterns per agent
Each agent has a specific relationship to Foundry IQ. This is the argument that makes the integration non-removable.

DocumentAnalyzer sends one query per topic to Foundry IQ with knowledge_source_ids scoped to each document individually. The return is a per-document array of grounded answers — each with source_document, section, passage, and confidence. This is what makes multi-source comparison possible: you get 7 answers to the same question, each attributed. Without Foundry IQ, you concatenate all documents into one prompt and lose per-source attribution. At real enterprise scale (50+ policies), you also blow the context window.

python
# DocumentAnalyzer — per-topic, per-document grounded retrieval
def extract_per_document(self, topic: str, documents: list[KnowledgeSource]) -> list[PolicyStatement]:
    results = []
    for doc in documents:
        response = foundry_iq.query(
            query=f"What rule or policy applies to: {topic}?",
            knowledge_source_ids=[doc.id],
            require_grounded_citations=True,
        )
        if response.has_content:
            results.append(PolicyStatement(
                document=doc.name,
                section=response.citation.section,
                passage=response.citation.exact_text,
                confidence=response.confidence,
                topic=topic,
            ))
    return results
ConflictDetector receives the list of PolicyStatement objects and applies a contradiction-detection prompt. The prompt structure is: "Here are N statements about [topic], each from a different authoritative source. Identify any pair that makes incompatible claims. Explain why they cannot simultaneously be satisfied for the same population of employees." This is distinct from single-document analysis — the agent is comparing authoritative answers, not re-reading documents.

ImpactAssessor sends a follow-up Foundry IQ query: "Which employee groups, systems, or teams are mentioned in connection with [conflict topic]?" This grounds the affected-employee count in actual policy text rather than in LLM inference.

RiskQuantifier queries Foundry IQ with regulatory context: "What are the penalties for [regulation name] non-compliance?" The knowledge base should include the actual regulation text (DPDP Act §91, FCA handbook excerpts) alongside the company policies so the risk estimates are grounded, not hallucinated.

ResolutionRecommender queries: "What resolution approaches exist for [conflict type] in [regulatory context]?" This grounds resolution suggestions in regulatory best practice, not just LLM pattern matching.

The ablation test result (document this in README): removing Foundry IQ and replacing with single-context concatenation produces (a) no section-level citations, (b) LLM paraphrases rather than exact policy language, (c) context overflow at ~50,000 tokens for real enterprise corpora, and (d) outputs that cannot be shown to a regulator. The Foundry IQ path produces auditable findings. That is the exact word to use: auditable.

Exact agent prompts
DocumentAnalyzer system prompt:

You are a policy document analyst. You receive grounded citations retrieved from a single enterprise policy document. Your task: extract the specific rule or constraint that applies to the given topic. Return ONLY the exact claim made by this document — do not infer, extrapolate, or summarize. If the document is silent on the topic, say so explicitly. Return JSON: {statement: string, scope: string, effective_date: string, is_silent: boolean}
ConflictDetector system prompt:

You are a policy conflict analyst. You receive N policy statements about the same topic, each from a different authoritative source document with a Foundry IQ citation. Your task: identify any pair or group of statements that make incompatible claims — claims that cannot simultaneously be satisfied for the same employee population. Explain, in plain language, WHY the conflict is logically impossible — not just that the statements differ. If a conflict requires a specific employee scenario to be impossible (e.g. Indian-resident remote workers), specify that scenario precisely. If no genuine conflict exists, say so. Return JSON: {has_conflict: boolean, conflict_pairs: [...], reasoning: string, severity: CRITICAL|HIGH|MEDIUM, confidence: float}
Key constraint: confidence below 0.65 → output status: UNCERTAIN, do not classify. The UI routes these to a separate human review queue, never the main dashboard.

Reliability implementation — exact code architecture

class ConflictSenseOrchestrator:
    PRIMARY_MODEL = "gpt-4o"          # via Azure Foundry
    BACKUP_MODEL  = "gpt-4o-mini"     # via Azure Foundry
    
    CONFIDENCE_THRESHOLDS = {
        "high":   (0.85, 1.00),   # auto-classify, show in dashboard
        "medium": (0.65, 0.85),   # show with REVIEW REQUIRED badge
        "low":    (0.00, 0.65),   # route to human review queue only
    }

    def call_agent(self, agent: str, payload: dict) -> dict:
        # Tier 1 — Primary
        try:
            result = self._call(self.PRIMARY_MODEL, payload, timeout=10)
            self._validate(result)
            return result
        except (APIError, TimeoutError, ValidationError) as e:
            self._log(f"PRIMARY_FAILED:{agent}:{e}")
        
        # Tier 2 — Backup
        try:
            result = self._call(self.BACKUP_MODEL, payload, timeout=15)
            self._validate(result)
            result["_meta"]["fallback"] = "TIER_2"
            return result
        except (APIError, TimeoutError, ValidationError) as e:
            self._log(f"BACKUP_FAILED:{agent}:{e}")
        
        # Tier 3 — Pre-computed mock (demo always completes)
        self._log(f"MOCK_ACTIVATED:{agent}")
        return self.mock_responses[agent]

    def _validate(self, result: dict) -> None:
        """Block any conflict finding without ≥2 Foundry IQ citations."""
        if result.get("has_conflict"):
            citations = result.get("citations", [])
            if len(citations) < 2:
                raise ValidationError("Conflict requires ≥2 grounded citations")
            for c in citations:
                if not all(k in c for k in ["source", "section", "confidence"]):
                    raise ValidationError(f"Malformed citation: {c}")
