# Screenshot Checklist

## Must-Have Screenshots
- Landing page with the Nexora policy set loaded
- SSE analysis in progress with live reasoning trace
- Conflict card showing the anonymity contradiction
- Citation hover or evidence panel showing grounded source text
- Human Approval Gate with approve, reject, and escalate actions visible
- Mock mode indicator, if the fallback path is triggered

## Best Supporting Evidence
- Foundry IQ citation trace or retrieval output
- Terminal showing backend and frontend running
- Any validation state that proves weak findings are filtered out
- Dashboard view showing the final, readable conflict summary

## Composition Guidance
- Capture one screen that explains the product in 3 seconds.
- Capture one screen that proves grounding.
- Capture one screen that proves human control.
- Capture one screen that proves reliability.

## What to Avoid
- Empty loading states.
- Overscrolled developer logs.
- Screenshots that show only JSON.
- Anything that hides the reasoning trace or the evidence.

## Caption Ideas
- "Grounded policy contradiction detected"
- "Reasoning trace streamed live"
- "Human approval required before action"
- "Fallback mode preserved demo reliability"
