# Foundry IQ Usage

## Why Foundry IQ Matters
ConflictSense depends on Foundry IQ to keep the system grounded, auditable, and section-aware.

## How It Is Used
### DocumentAnalyzer
- Queries one source document at a time.
- Requests grounded citations for every answer.
- Preserves section-level attribution.

### ConflictDetector
- Uses grounded outputs from multiple documents to compare policy statements.
- Relies on evidence, not paraphrase.

### ImpactAssessor
- Retrieves entities, teams, systems, and affected groups from grounded sources.
- Avoids invented scope or impact assumptions.

## What Foundry IQ Enables That Generic RAG Does Not
- Better citation fidelity.
- Less context-window pressure.
- Cleaner attribution to the exact source passage.
- A more defensible audit trail.

## Judge-Ready Framing
Foundry IQ is not just a backend dependency. It is the mechanism that makes the product trustworthy enough to show to a compliance-minded enterprise audience.

## Key Line for the Demo
Every finding in ConflictSense is backed by grounded evidence, so the user can trace the output back to the source text instead of trusting the model blindly.

## Why This Is Important
This is the difference between "the model thinks" and "the system can be audited."
