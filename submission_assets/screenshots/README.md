# Screenshot Strategy

## 1. Landing Page
- Purpose: Establish the product and the enterprise context instantly.
- Exact UI state: Home screen with Nexora loaded and no analysis running.
- Why judges care: It shows polish and makes the demo feel real before anything happens.
- Suggested filename: `01_landing_page.png`
- Suggested caption: `ConflictSense loads the enterprise policy set`
- Judging criterion supported: Demo Quality, Enterprise Value, Memorability

## 2. Live Reasoning Trace
- Purpose: Prove the product is reasoning in public, not hiding behind JSON.
- Exact UI state: Analysis in progress with multiple SSE messages visible.
- Why judges care: This is the strongest evidence for multi-step thinking.
- Suggested filename: `02_live_reasoning_trace.png`
- Suggested caption: `The reasoning trace streams step by step`
- Judging criterion supported: Innovation, Technical Execution

## 3. Conflict Detected
- Purpose: Show the core capability working.
- Exact UI state: Conflict card expanded with the anonymity contradiction visible.
- Why judges care: It proves the system finds a structural impossibility, not a keyword match.
- Suggested filename: `03_conflict_detected.png`
- Suggested caption: `Anonymity guarantee conflicts with identity logging`
- Judging criterion supported: Enterprise Value, Innovation

## 4. Conflict Rejected
- Purpose: Show trust and validation behavior.
- Exact UI state: A low-quality or weak candidate marked rejected by validation.
- Why judges care: Judges want to see the system can say no to bad output.
- Suggested filename: `04_conflict_rejected.png`
- Suggested caption: `Weak findings are rejected before they reach the dashboard`
- Judging criterion supported: Responsible AI, Technical Execution

## 5. Impact Assessment
- Purpose: Show business relevance beyond the contradiction itself.
- Exact UI state: Impact panel listing affected employees, systems, or teams.
- Why judges care: It translates an abstract conflict into enterprise risk.
- Suggested filename: `05_impact_assessment.png`
- Suggested caption: `The impacted scope is grounded and visible`
- Judging criterion supported: Enterprise Value, Innovation

## 6. Approval Gate
- Purpose: Show human control and governance.
- Exact UI state: Approval Gate visible with approve, reject, and escalate actions.
- Why judges care: It proves the system is safe to trust in enterprise settings.
- Suggested filename: `06_approval_gate.png`
- Suggested caption: `Human approval is required before action`
- Judging criterion supported: Responsible AI, Enterprise Value

## 7. Mock Mode Fallback
- Purpose: Prove the demo is resilient.
- Exact UI state: `[MOCK MODE]` label visible in the UI.
- Why judges care: It removes the biggest risk in live judging: runtime failure.
- Suggested filename: `07_mock_mode_fallback.png`
- Suggested caption: `Fallback mode preserved the demo without hiding it`
- Judging criterion supported: Demo Quality, Technical Execution

## 8. Final Dashboard
- Purpose: Show the end state after validation and human review.
- Exact UI state: Completed dashboard with final status and citations visible.
- Why judges care: It closes the loop and makes the product feel finished.
- Suggested filename: `08_final_dashboard.png`
- Suggested caption: `The final dashboard presents a validated, auditable outcome`
- Judging criterion supported: Demo Quality, Enterprise Value

## Capture Standard
Every screenshot should answer three questions at a glance:
- What happened?
- Why does it matter?
- Can a judge trust it?
